Project information

  • Technologies: In-house MVC framework, PHP, HTML, CSS, JavaScript, AJAX, MySQL, CKEditor, Cron Jobs (No PHPMailer used)

Overview

I developed an internal Email Reminder System to help all staff — including accounting, sales, and general team members — schedule emails in advance. Before this system, staff often forgot to send important follow-up emails, leading to customer dissatisfaction. This tool lets users pre-write messages and set a future date for sending. It includes a "Cron" function that reuses the same email content — allowing quick edits to client info like email addresses, improving efficiency and accuracy.

Challenges Solved:

  • First time integrating CKEditor for rich-text email content creation.
  • Built the system without relying on third-party tools like PHPMailer.
  • Designed a simple interface so non-technical staff can schedule emails easily.
  • Reduced email mistakes caused by forgotten manual follow-ups.

Key Outcomes:

  • Helped accounting and sales teams stay consistent with customer follow-ups.
  • Simplified email setup with reusable message content — users only need to change the recipient’s address.
  • Cron-based scheduling removed the need for manual email sending.
